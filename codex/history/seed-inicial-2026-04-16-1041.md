# Seed Inicial

## Contexto

O projeto precisava de um mecanismo simples para popular dados iniciais de desenvolvimento sem depender de insercoes manuais no MongoDB.

## Decisão

Foi criado um seed operacional em `src/lib/db/seeds`, com datasets tipados para contas e categorias padrao, mais uma opcao `--with-sample-data` para gerar transacoes, orçamento e meta de exemplo do mes atual. A execucao ficou exposta por `npm run seed`.

## Impacto

O projeto passa a ter bootstrap de dados previsivel e idempotente para desenvolvimento local. O script nao duplica registros ja existentes, reutiliza a conexao central do MongoDB e carrega `.env.local` automaticamente via `@next/env`.

Para suportar o uso do mesmo helper pelo app e pelo script CLI, `src/lib/db/connect.ts` deixou de importar `server-only` diretamente. A fronteira server-side continua protegida nos repositories e services que usam a conexao dentro do app.
