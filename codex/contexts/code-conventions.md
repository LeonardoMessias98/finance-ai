# Code Conventions

## Convenções gerais

- usar TypeScript estrito
- não usar `any`
- manter funções pequenas
- usar nomes explícitos
- preferir arquivos enxutos

## Convenções de componentes

- preferir Server Components
- usar Client Components apenas quando houver necessidade real
- extrair componentes compartilhados para `src/components`

## Convenções de domínio

- usar Zod para validação
- usar React Hook Form em formulários
- usar Mongoose para persistência
- usar Server Actions quando houver mutações
